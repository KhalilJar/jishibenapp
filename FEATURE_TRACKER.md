## 本轮实现细节

### 数据层
- Room 数据库升级到 `version 2`
- `records` 表新增 `accountId`
- 新增 `accounts`、`budgets`、`recurring_rules` 表
- 旧记录迁移后自动归到默认账户“现金”
- 默认账户内置：现金、银行卡、支付宝、微信

### 记录能力
- 复用新增记录页作为编辑页
- 支持修改：收支类型、金额、标签、备注、日期、账户
- 删除记录前增加二次确认
- 编辑/删除后，记录列表、日历、统计会同步刷新

### 筛选与统计
- 记录页支持：备注关键字、收支类型、标签、账户、日期范围、金额范围筛选
- 日历页与统计页新增账户筛选
- 统计页扇形图与明细会跟随时间粒度、收支类型、账户条件联动

### 预算与固定收支
- 支持设置“本月总支出预算”
- 支持设置“某标签月预算”
- 超预算时在记录页显示提醒
- 固定收支第一版只做提醒，不自动静默入账
- 到期规则可一键带出预填表单“记一笔”

### 数据管理
- 备份文件包含：记录、账户、预算、固定收支规则
- 备份为单文件 JSON，再经过密码加密后导出
- 导入时需要同一密码解密恢复
- 当前阶段不做多设备实时同步

## 外部调研结论
- 成熟记账产品普遍重视预算、固定收支、搜索筛选、账户维度管理
- 对当前这个 App 来说，优先把“本地数据安全 + 管理效率”做扎实，比过早上复杂后端更划算
- 手机上的统计与图表继续使用 Kotlin / Compose 最合适，不需要引入 Python
